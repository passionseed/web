Objective:
I want to improve the UI/UX and code quality of a Next.js component. Help me go through a design improvement workflow, covering both visual and structural aspects.

Step 1: 📸 Current Component Overview
Prompt:
"Here is my current component (code + screenshot if possible). Please review the design from both UI and code perspectives. Identify any obvious flaws or areas for improvement."

Step 2: 🎯 Define Purpose and Users
Prompt:
"The goal of this component is to [briefly explain purpose]. It is used by [user type]. How can the design better align with the user's goals and usage context?"

Step 3: 🎨 UI Improvement Suggestions
Prompt:
"Based on modern UI/UX principles (e.g. consistency, clarity, responsiveness), suggest improvements in layout, colors, spacing, and interactions."

Optional: Provide TailwindCSS/Framer Motion examples.

Step 4: ⚙️ Component Structure & Code
Prompt:
"Review the component’s code structure. Are there better ways to organize props, states, or reusable parts? Suggest changes to improve maintainability and performance."

Step 5: 🔁 Variants and Responsiveness
Prompt:
"What are possible states (loading, empty, error, interaction)? Suggest how to handle and design for them. Also, how can this component adapt well to mobile or different screen sizes?"

Step 6: 🧪 Test Cases & Edge Scenarios
Prompt:
"What test cases should I consider to ensure this component behaves well in production? Suggest examples that might break or look weird."

Step 7: ✨ Final Touches
Prompt:
"Suggest any polish or micro-interactions (e.g., hover, animation) that would make the experience feel more refined."
